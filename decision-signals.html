Confidence Scoring

"Did a human intentionally make this change?" — not whether it matters (that's importance). High confidence = deliberate decision, not API glitch or auto-optimization.

High confidence (0.85–0.99)
"Almost certain a human did this"
Campaign paused at 2pm Tuesday — clearly manual
Low confidence (0.50–0.70)
"Can't be sure it was intentional"
Bid changed — could be automated rules

Scoring Factors (weighted → 0.0–1.0)

FactorWeightHigh signalLow signal
Field specificity25%campaign.status (always manual)metrics.impressions (not a decision)
Change pattern20%Round numbers ($100→$200)Fractional (bid 1.23→1.24 = algo)
Time-of-day15%Business hours, weekday3am Sunday = automated rule
Correlated changes15%Budget + bid + status togetherSingle micro-adjustment = algo
Platform automation15%No active rules for entityAuto-bidding / CBO active
Change velocity10%Single discrete change = clickGradual increment = algorithmic

Importance Scoring

Every detected decision gets an importance level (CRITICAL / HIGH / MEDIUM / LOW) based on contextual signals. Helps teams prioritize which to review first.

Scoring Factors

FactorWeightWhat it measuresExamples
Spend at Risk30%Budget affected by this change$400/day = high; $15/day = low
Mid-flight Disruption25%Was entity active & spending?Paused while spending = critical
Change Magnitude20%% or absolute size+100% budget = high; +10% bid = low
Learning Phase Trigger15%Resets optimization model?Bid strategy switch = resets
Destructive vs Additive10%Removes or adds?Pausing = destructive; keyword add = additive

How Signal Uses Change Reports

1.Ingest — Pull from Google Change History & Meta Activity Log every hour 2.Normalize — Map into unified schema (entity_type, field, old/new) 3.Filter — Separate humans (Web UI / named) from automated (API / bots) 4.Classify — Map to one of 16 decision types by field + operation 5.Score — Calculate confidence + importance 6.Present — Surface in Decision Tracker for confirmation

What the customer gets, when — before the decision reservoir exists. No empty dashboards, no "check back in a quarter."

"It's hard to sell something when the customer expects value only after a year." — The reframe: day-0 value comes from connecting data + listener + existing models.

NO DATA NEEDED · PUBLIC SOURCES ONLY

3.Connect — Auth Google Ads + Meta. APIs return 30 days on first call.
