What We Listen To

For each of the 16 decision types, here's exactly which API fields Signal monitors, how we detect that a decision was made, and what we store in the decision journal.

16 Decision Types

BUDGET_SET BUDGET_REALLOCATE BID_STRATEGY_CHANGE BID_ADJUST PACING_CHANGE AUDIENCE_TARGET AUDIENCE_EXPAND AUDIENCE_SUPPRESS KEYWORD_MANAGE PLACEMENT_MANAGE GEO_TARGET SCHEDULE_DAYPART CREATIVE_MANAGE STATUS_CHANGE FREQUENCY_CAP SIGNAL_CONFIG
BUDGET_SET Set, raise, or lower budget (daily or lifetime) at campaign/ad-set level
Primary detection fields
campaign_budget.amount_micros campaign_budget.type (DAILY/LIFETIME) adset.daily_budget adset.lifetime_budget campaign_budget.period
Supporting context fields
metrics.cost_micros (pre/post) metrics.impressions (pre/post) campaign.name adset.name
Detection logic: If amount_micros at time T differs from T-1 by more than the noise threshold (±5%), classify as BUDGET_SET. Direction (increase/decrease) and magnitude stored. If multiple ad sets change in opposite directions on the same day within the same campaign → reclassify as BUDGET_REALLOCATE instead.
BUDGET_REALLOCATE Shift budget between campaigns/ad sets within a platform
Primary detection fields
adset.daily_budget (multiple ad sets) campaign_budget.amount_micros (multiple campaigns) adset.campaign_id
Supporting context fields
campaign.name adset.name metrics.cost_micros
Detection logic: Two or more budget changes on the same day where the net total is approximately constant (within 10%). One goes up, another goes down → this is a reallocation, not independent budget sets. We store both source and destination entities.
BID_STRATEGY_CHANGE Switch strategy (manual ↔ auto; tCPA/tROAS/cost-cap/max-conv)
Primary detection fields
campaign.bidding_strategy_type campaign.target_cpa.target_cpa_micros campaign.target_roas.target_roas adset.bid_strategy (Meta) adset.billing_event
Supporting context fields
metrics.average_cpc (before/after) metrics.conversions (before/after) metrics.cost_per_conversion
Detection logic: bidding_strategy_type enum value changes between snapshots. We record from→to strategy, and any associated target value. Context metrics help assess if the switch improved performance.
BID_ADJUST Change bid amount, modifiers, or bid factors
Primary detection fields
ad_group.cpc_bid_micros ad_group_criterion.cpc_bid_micros campaign.target_cpa.target_cpa_micros adset.bid_amount campaign_criterion.bid_modifier
Detection logic: Bid values or bid modifiers change by >5% between snapshots without a strategy-type change. This is a tuning decision within the same strategy — distinct from BID_STRATEGY_CHANGE.
PACING_CHANGE Standard ↔ accelerated; toggle auto-allocator
Primary detection fields
campaign_budget.delivery_method adset.pacing_type campaign.campaign_budget_optimization (CBO toggle) adset.budget_rebalance_flag
Detection logic: delivery_method changes (STANDARD↔ACCELERATED) or CBO/ABO toggle flips. Also detects enabling Advantage Campaign Budget on Meta.
AUDIENCE_TARGET Add/remove targeting (demo, interest, custom, lookalike, retargeting)
Primary detection fields
adset.targeting.custom_audiences adset.targeting.lookalike_spec adset.targeting.interests campaign_criterion.user_list ad_group.audience_setting adset.targeting.age_min/max adset.targeting.genders
Detection logic: Audience arrays/lists differ between snapshots — new IDs appear or existing ones vanish. We diff the audience object and record adds vs removes, plus the audience type (LAL, custom, interest, demographic).
AUDIENCE_EXPAND Broaden reach beyond defined segment
Primary detection fields
adset.targeting.targeting_optimization adset.targeting.expansion_enabled campaign.audience_expansion_opt_in adset.targeting.flexible_spec
Detection logic: Expansion flag toggles on, or flexible_spec broadens (interest targeting opens up). Distinguished from AUDIENCE_TARGET by checking if it's an expansion setting vs a specific audience add.
AUDIENCE_SUPPRESS Exclude a segment from delivery
Primary detection fields
adset.targeting.excluded_custom_audiences campaign_criterion.negative (user_list) adset.targeting.exclusions campaign.brand_safety_negative_keywords
Detection logic: New entries appear in exclusion lists. Any audience added to the excluded_custom_audiences array or a negative user list criterion added to a campaign.
KEYWORD_MANAGE Add/remove keywords, match types, negatives (search surfaces)
Primary detection fields
ad_group_criterion.keyword.text ad_group_criterion.keyword.match_type ad_group_criterion.status ad_group_criterion.negative campaign_criterion.keyword (campaign negatives) shared_set.shared_set_id (negative lists)
Detection logic: New keyword criteria appear or existing ones change status (ENABLED↔REMOVED↔PAUSED). We count adds vs removes, classify by match type, and flag if they're negative keywords (blocking) vs positive (targeting).
PLACEMENT_MANAGE Inventory/placement targeting, exclusions, deals/PMP
Primary detection fields
adset.targeting.publisher_platforms adset.targeting.facebook_positions adset.targeting.instagram_positions campaign_criterion.placement campaign_criterion.youtube_channel campaign.network_settings
Detection logic: Placement arrays change — platforms added/removed, specific positions toggled. On Google: network settings or placement criterion updates. On Meta: publisher_platforms or position arrays differ.
GEO_TARGET Location, radius, geo bid factors
Primary detection fields
campaign_criterion.location adset.targeting.geo_locations campaign_criterion.proximity campaign_criterion.bid_modifier (location) adset.targeting.excluded_geo_locations
Detection logic: Location criteria added/removed, radius targeting changes, or geo-level bid modifiers adjust. We store the specific locations and whether they were adds, removes, or modifier changes.
SCHEDULE_DAYPART Flight dates, dayparting
Primary detection fields
campaign.start_date campaign.end_date campaign_criterion.ad_schedule adset.start_time adset.end_time
Detection logic: Flight dates change, ad schedule criteria appear/modify, or start/end times on Meta ad sets shift. Daypart bid modifiers changing also triggers this.
CREATIVE_MANAGE Upload/swap/rotate/enable/pause assets; DCO config
Primary detection fields
ad.creative.id ad.status adcreative.object_story_spec adcreative.video_id adcreative.image_hash ad_group_ad.ad.responsive_search_ad asset_group.listing_group_filter
Detection logic: New creative IDs appear in an ad set, ad status toggles, or creative assets change (new video_id, new image_hash). We track the count of adds/removes/pauses and the creative format (video, image, carousel, RSA).
STATUS_CHANGE Pause/enable/archive campaign, ad group, or ad
Primary detection fields
campaign.status ad_group.status ad_group_ad.status adset.effective_status adset.configured_status ad.status
Detection logic: Status enum changes (ENABLED↔PAUSED↔REMOVED). We record entity level (campaign/ad group/ad), direction (pause vs enable), and context (spend at time of pause, days since last conversion).
FREQUENCY_CAP Set impression frequency cap
Primary detection fields
adset.frequency_control_specs.event adset.frequency_control_specs.max_frequency adset.frequency_control_specs.interval_days campaign.frequency_caps
Detection logic: max_frequency or interval_days values change, or frequency_control_specs appear for the first time. We store before/after values and the event type (IMPRESSIONS usually).
SIGNAL_CONFIG Which conversion events to optimize toward; feed/CAPI/audience inputs
Primary detection fields
conversion_action.status conversion_action.category campaign.conversion_goals adset.promoted_object adset.optimization_goal pixel.event_config actions.action_source (server vs website)
Detection logic: Conversion action linked/unlinked from a campaign, optimization goal changes on Meta ad sets, or CAPI event sources appear. This captures changes to what the platform optimizes toward, not the media settings themselves.

Decision Journal — Stored Fields Per Entry

Every confirmed decision is stored with these fields, creating the structured history Signal learns from.

FieldTypeDescription
decision_idUUIDUnique identifier for this decision event
detected_attimestampWhen Signal detected the change
occurred_atdateWhen the change actually happened (from field diff)
decision_typeenumOne of the 16 types (BUDGET_SET, BID_STRATEGY_CHANGE, etc.)
platformenumGOOGLE_ADS | META | DV360 | TIKTOK
entity_typeenumCAMPAIGN | AD_GROUP | AD_SET | AD | KEYWORD
entity_idstringPlatform-specific ID of the changed entity
entity_namestringHuman-readable name at time of detection
field_changesJSON[]Array of {field, old_value, new_value} for each changed field
confidencefloat0.0–1.0, how confident we are this was intentional
statusenumPENDING | CONFIRMED | REJECTED
intenttextFree-text explanation from the user (why they made this change)
intent_categoryenumAI-classified: GROWTH | EFFICIENCY | TESTING | SEASONAL | REACTIVE | OTHER
context_metricsJSONPerformance metrics at time of change (CPA, ROAS, spend, conversions)
outcome_metricsJSONPerformance metrics 7 days after change (filled retroactively)
user_idstringWho confirmed/rejected (for multi-user accounts)
account_idstringPlatform account identifier
